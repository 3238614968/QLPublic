# QLPublic

## 项目介绍

这是一个面向青龙（QL）/本地定时任务场景的脚本仓库，主要用于自动化执行日常签到任务与辅助流程（如验证码打码、通知推送）。目前仓库包含 Python 与 Node.js 两类脚本，适配不同平台与服务的签到需求。

## 目录结构

- `品赞签到.py`：品赞 HTTP 代理签到脚本，支持多账号登录与签到。
- `zhenxun_2captcha.js`：臻讯云自动登录/签到脚本，集成 2Captcha 处理腾讯防水墙验证码。

## 适用场景

- 在青龙面板中通过环境变量集中管理账号并定时执行签到。
- 在本地服务器/开发机中通过 cron 或任务计划实现自动化运行。
- 需要对多账号签到结果进行统一日志输出与通知。

## 功能说明

脚本会按以下流程执行：

1. 从环境变量 `pzhttp` 读取账号配置。
2. 对每个账号执行登录，获取 token。
3. 使用 token 调用签到接口。
4. 输出每个账号的执行结果与汇总信息。

## 环境变量配置

变量名：`pzhttp`

格式：`账号#密码`，多账号用换行分隔。

示例：

```text
13800138000#MyPass123
13900139000#AnotherPass456
```

## 运行方式

### 本地运行

1. 安装 Python 3.8+。
2. 安装依赖：

```bash
pip install requests
```

3. 配置环境变量并运行：

```bash
export pzhttp=$'13800138000#MyPass123\n13900139000#AnotherPass456'
python 品赞签到.py
```

### 青龙面板

1. 新建环境变量 `pzhttp`，按上面的格式填写。
2. 新建定时任务执行：

```bash
python 品赞签到.py
```

## 输出说明

- `登录失败`：账号密码错误、风控、或接口返回失败信息。
- `签到失败`：登录成功但签到接口返回失败。
- `登录请求异常 / 签到请求异常`：网络问题、超时、依赖缺失等。

## 注意事项

- 请勿在公开场景泄露账号密码。
- 接口属于第三方服务，若页面或接口策略调整，脚本可能需要同步更新。
- 如环境缺少 `requests`，脚本将无法运行。
